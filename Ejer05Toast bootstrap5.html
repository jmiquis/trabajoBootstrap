CAROUSEL



El plugin "carousel" , más conocido como slideshow permite presentar una serie de fotografias que van pasando de una a otra de manera automática o manual

El contenedor general tiene que pertenecer obligatoriamente a la clase "carousel slide" y llevar el atributo data-bs-ride ="carousel"

<div id="carouselExampleIndicators" class="carousel slide" data-bs-ride="carousel">

Dentro de este div general hay otro con la clase "carousel-inner"

<div class="carousel-inner">

Ahora si, dentro de este van los div contenedores de cada una de las imágenes que queremos añadir al slide. Cada uno de ellos deberá llevar la clase "carousel-item" y la que queramos mostrar la primera deberá ser de la clase "carousel-item active" Una vez creados estos div deberán llevar una etiqueta img con su src correspondiente y es una buena práctica ponerles a las etiquetas img la clase "d-block w-100" para prevenir la alineación automática de cada navegador

<div class="carousel-item active">

<img src="../Imagenes/cafe-burst.jpg" class="d-block w-100" alt="...">

Si se quieren agregar indicadores, como en este ejemplo, se deben utilizar botones con la clase "carousel-control-prev" y "carousel-control-next" respectivamente, cada uno con dos etique span, una con la clase "carousel-control-prev-icon" y la etiqueta aria-hidden y el otro con la clase "visually-hidden"

<button class="carousel-control-prev" type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide="prev">

<span class="carousel-control-prev-icon" aria-hidden="true"">

<span class="visually-hidden">

Si queremos poner selectores de imagen para ir directamente a una imagen determinada, abrimos un div con la clase "carousel-indicators" y el id = "ejemploIndicadores", con un boton para cada imagen y las etiquetas data-bs-target="#ejemploIndicadores" y data-bs-slide-to = al numero de imagen que enlaza

<div class="carousel-indicators">

<button type="button" data-bs-target="#carouselExampleIndicators" data-bs-slide-to="0" class="active" aria-current="true" aria-label="Slide 1">

POPOVER



Este componente se muestra cuando el usuario hace clic en un elemnento. Es similar al tooltip, pero se muestra más contenido en este caso.


Cosas que debe saber al usar el complemento popover: Los popovers se basan en la biblioteca de terceros Popper.js para el posicionamiento. ¡Debe incluir popper.min.js antes de bootstrap.js o usar bootstrap.bundle.min.js/ bootstrap.bundle.jsque contiene Popper.js para que funcionen los popovers!
Los popovers requieren el complemento de información sobre herramientas como una dependencia. Si está compilando nuestro JavaScript desde la fuente, requiereutil.js .
Los popovers son opcionales por motivos de rendimiento, por lo que debe inicializarlos usted mismo .
La longitud cero titley contentlos valores nunca mostrarán un popover.
Especifique container: 'body'para evitar problemas de representación en componentes más complejos (como nuestros grupos de entrada, grupos de botones, etc.).
Activar popovers en elementos ocultos no funcionará.
Los popovers para elementos .disabledor disableddeben activarse en un elemento contenedor.
Cuando se activan desde anclas que se envuelven en varias líneas, los popovers se centrarán entre el ancho total de las anclas. Úselo white-space: nowrap;en sus enlaces para evitar este comportamiento.
Los popovers deben ocultarse antes de que sus elementos correspondientes se eliminen del DOM.

Ejer05: Bootstrap -Toast-


Toast es mensaje de alerta ligero y fácilmente personalizable, con el cual podemos, opcionalmente, enviar notificaciones a los usuarios/visitantes de una web. Este se muestra por un par de segundos al producirse un evento (ej: clicar botones, enviar formularios...ect).


Toast admite múltiples personalizaciones o configuraciones:

  • CREACION: Para crear un Toast necesitamos usar una clase: .toastclase y dentro de ella podemos dividir la estructura en la cabecera usando .toast-headery y en cuerpo usando .toast-bodydentro.
  • ESTRUCTURA: Dentro de la clase creada podemos dividir la estructura en la cabecera usando .toast-headery y en cuerpo usando .toast-bodydentro.
  • MOSTRAR Y OCULTAR: Por defecto los Toast están ocultos de forma predeterminada, por lo que para mostrarlos necesitamos usar la clase .showclase. Por otra parte se cierra automáticamente al cabo de un par de segundos, aunque si quisiéramos personalizarlo para poder ser cerrado por un usuario, requerimos de un botón, al que debebemos agregar: data-bs-dismiss="toast".
  • HUBICACION: Toast puede situarse donde uno quiera y cuantos uno quiera o necesite; Toast, admite configuración con Flex box.
  • COLOR: los esquemas de Toast están divididos en base a color y color de fondo; incluso admite transparencias.
  • ESTILO: podemos darle cualquier estilo que queramos y podemos incluir dentro con iconos o vectores.
  • ACCESIBILIDAD: la estructura de Toast deben estár envueltos en una aria-liveregión, ya que su principal objetivo es comunicativo.
